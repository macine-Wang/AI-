# 智能简历筛选模块 - 详细功能说明

## 📋 功能概述

智能简历筛选模块是一个基于AI的简历管理系统，支持OCR识别、智能解析、多维度评分、灵活筛选等功能。

---

## ✨ 核心功能

### 1. 简历上传与解析 📄

#### 1.1 多种上传方式

**支持的上传方式：**
- ✅ 单个文件上传
- ✅ 批量拖拽上传（一次可上传多份）
- ✅ 支持压缩包批量导入
- 🔄 从招聘平台直接导入（需集成API）

**支持的文件格式：**
- PDF（`.pdf`）
- Word文档（`.doc`, `.docx`）
- 图片格式（`.jpg`, `.png`）

#### 1.2 OCR文档识别

**技术方案：**
- 百度AI OCR（推荐）- 中文识别准确率98%+
- 腾讯云OCR - 识别速度快
- 阿里云OCR - 企业级稳定

**识别流程：**
```
PDF/图片 → OCR识别 → 提取文本 → AI解析 → 结构化数据
```

**置信度评估：**
- 整体置信度：0-100分
- 各字段置信度独立评分
- 低置信度字段标记，提示人工校验

#### 1.3 AI智能解析

**自动提取的信息：**

**基础信息：**
- 姓名
- 性别
- 年龄
- 联系电话
- 电子邮箱
- 现居地
- 期望薪资
- 当前薪资

**教育背景：**
- 毕业院校
- 专业
- 学历（博士/硕士/本科/专科）
- 毕业时间
- GPA/成绩排名
- 院校类型（985/211/双一流/普通本科）

**工作经历：**
- 公司名称
- 职位名称
- 工作时间（起止日期）
- 工作内容描述
- 主要业绩成果
- 是否为知名大公司

**项目经验：**
- 项目名称
- 担任角色
- 项目时间
- 技术栈
- 项目描述
- 项目成果

**技能信息：**
- 专业技能列表
- 语言能力（如：英语CET-6）
- 证书资质

**其他信息：**
- 自我评价
- 个人优势
- 获奖情况
- 求职意向
- 到岗时间
- 工作年限（自动计算）

#### 1.4 解析质量保障

**质量评估指标：**
- **整体置信度**：综合评分 0-100
- **字段置信度**：每个字段独立评分
- **缺失字段提示**：列出未能提取的关键信息
- **警告信息**：标注可能存在的问题

**人工校正入口：**
- 低置信度字段高亮显示
- 一键进入编辑模式
- 快速修正错误信息
- 保存校正历史，优化AI模型

### 2. 字段管理与自定义 🏷️

#### 2.1 标准字段库

**预设筛选字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 学历 | 枚举 | 博士/硕士/本科/专科/高中 |
| 院校类型 | 枚举 | 985/211/双一流/普通本科/专科 |
| 工作年限 | 数字 | 自动计算总工作时长 |
| 年龄 | 数字 | 根据出生日期计算 |
| 期望薪资 | 数字 | 单位：元/月 |
| 当前薪资 | 数字 | 单位：元/月 |
| 所在城市 | 文本 | 当前居住城市 |
| 期望城市 | 文本 | 期望工作城市 |
| 英语水平 | 文本 | CET-4/6, 雅思, 托福等 |
| 当前职位 | 文本 | 最近一份工作的职位 |
| 行业背景 | 文本 | 所在行业 |
| 大厂经验 | 布尔 | 是否有BAT等大厂背景 |

#### 2.2 自定义字段（高级功能）

**创建自定义字段：**
- 字段名称
- 字段类型（文本/数字/布尔/枚举）
- 字段描述
- 默认值
- 是否必填

**AI辅助标注：**
- 根据简历内容自动填充
- 支持正则表达式匹配
- 关键词提取

**使用场景示例：**
- "是否有AI项目经验"
- "是否有创业经历"
- "管理团队规模"
- "专利数量"

### 3. 智能评分与排名 ⭐

#### 3.1 多维度评分系统

**五大评分维度：**

**1️⃣ 学历评分（0-100分）**
- 博士：95-100分
- 硕士：85-95分
- 本科985：80-90分
- 本科211：75-85分
- 普通本科：65-75分
- 专科：50-65分

**2️⃣ 经验匹配度（0-100分）**
- 工作年限匹配
- 岗位经验相关度
- 行业背景匹配
- 项目经验相关度
- 大厂背景加分

**3️⃣ 技能匹配度（0-100分）**
- 必备技能覆盖率
- 加分技能覆盖率
- 技能熟练程度
- 技术栈深度

**4️⃣ 稳定性评分（0-100分）**
- 平均每份工作时长
- 跳槽频率
- 职业发展连贯性
- 大厂稳定性

**5️⃣ 成长性评分（0-100分）**
- 职位晋升轨迹
- 薪资增长趋势
- 学习能力体现
- 创新能力

**综合评分计算：**
```
总分 = 学历分 × 20% + 
       经验匹配 × 25% + 
       技能匹配 × 35% + 
       稳定性 × 10% + 
       成长性 × 10%
```

#### 3.2 权重自定义

**不同岗位权重模板：**

**技术岗位：**
- 学历：20%
- 经验：25%
- 技能：35% ⬆️（重点）
- 稳定性：10%
- 成长性：10%

**管理岗位：**
- 学历：15%
- 经验：40% ⬆️（重点）
- 技能：20%
- 稳定性：15%
- 成长性：10%

**应届生：**
- 学历：40% ⬆️（重点）
- 经验：10%
- 技能：30%
- 稳定性：5%
- 成长性：15%

**保存评分模板：**
- 自定义模板名称
- 设置各维度权重
- 添加岗位要求
- 一键应用模板

#### 3.3 智能推荐理由

**自动生成推荐点：**
- ✅ "985高校毕业，学历背景优秀"
- ✅ "5年大厂经验，技术扎实"
- ✅ "技术栈完全匹配，可快速上手"
- ✅ "项目经验丰富，带过3人以上团队"

**风险点标注：**
- ⚠️ "跳槽频率较高，平均每份工作不足1年"
- ⚠️ "期望薪资涨幅较大（>50%）"
- ⚠️ "工作时间存在重叠，可能有兼职"
- ⚠️ "缺少相关行业经验"

### 4. 灵活筛选与排序 🔍

#### 4.1 单字段排序

**可排序字段：**
- 综合评分（默认）
- 学历高低
- 工作年限
- 年龄
- 期望薪资
- 上传时间

**排序方式：**
- 升序（ASC）
- 降序（DESC）

#### 4.2 多条件组合排序

**优先级排序：**
```
示例：
优先级1：学历（降序）
优先级2：工作年限（降序）
优先级3：年龄（升序）

结果：先按学历排，学历相同按工作年限排，工作年限相同按年龄排
```

**保存排序方案：**
- 命名排序规则
- 一键应用
- 方案管理

#### 4.3 高级筛选功能

**基础筛选：**
- 学历 ≥ 本科
- 工作年限 ≥ 3年
- 期望薪资 ≤ 20K
- 年龄 25-35岁

**标签筛选：**
- 已标记"重点关注"
- 已标记"待面试"
- 已标记"已淘汰"
- 自定义标签

**关键词搜索：**
- 全文搜索（姓名、技能、公司、职位）
- 高亮匹配关键词
- 支持模糊匹配

**复合条件：**
```
示例1：(本科 AND 3年以上) OR (硕士 AND 1年以上)
示例2：(985 OR 211) AND (大厂经验)
示例3：技能包含"React" AND 薪资≤25K
```

**筛选方案管理：**
- 保存筛选条件
- 命名筛选方案
- 快速应用方案
- 对比不同方案效果

```
AI推荐筛选条件：

- 有大厂经验优先
- 期望薪资 20K-35K
```

#### 6.2 异常提醒

**自动检测：**

**工作时间重叠：**
- 检测多份工作时间冲突
- 提示可能存在兼职或信息错误

**频繁跳槽：**
- 计算平均在职时长
- 标注跳槽频率过高（<1年/份）

**薪资异常：**
- 期望薪资涨幅>100%
- 提示期望可能过高

**学历疑问：**
- 毕业时间与年龄不符
- 学历层次跳跃异常

**工作经历异常：**
- 职位倒退
- 行业跨度过大

**筛选历史：**
- 保存每次筛选的条件
- 记录筛选结果数量
- 时间戳记录

**对比不同方案：**
- 方案A vs 方案B

**操作日志：**
- 简历上传记录

## 📊 使用场景示例

### 场景1：招聘前端开发工程师

**需求：**
- 本科及以上
- 3年以上工作经验
- 熟悉React/Vue
- 有大型项目经验
- 薪资预算20-30K

**筛选步骤：**

1. **上传简历**
   - 批量上传50份简历
   - 等待AI解析完成

2. **应用评分模板**
   - 选择"技术岗位标准模板"
   - 技能权重35%

3. **设置筛选条件**
   - 学历 ≥ 本科
   - 工作年限 ≥ 3年
   - 技能包含"React" OR "Vue"
   - 期望薪资 ≤ 30000

4. **排序**
   - 按综合评分降序

5. **筛选结果**
   - 从50份简历中筛选出15份
   - 重点关注90分以上的3份
   - 安排面试

---

### 场景2：校园招聘应届生

**需求：**
- 985/211高校
- 计算机相关专业
- 有实习或项目经验
- 学习成绩优秀

**筛选步骤：**

1. **应用评分模板**
   - 选择"应届生模板"
   - 学历权重40%

2. **设置筛选条件**
   - 学历 = 本科 OR 硕士
   - 院校类型 = 985 OR 211
   - 专业包含"计算机" OR "软件"
   - GPA ≥ 3.5

3. **排序**
   - 优先级1：院校类型（985优先）
   - 优先级2：GPA（降序）

4. **查看亮点**
   - 关注"有实习经验"
   - 关注"有项目经验"
   - 关注"有竞赛获奖"
